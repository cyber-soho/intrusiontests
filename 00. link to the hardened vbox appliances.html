How to Convert a VirtualBox VM to VMware Workstation Pro

Windows 10 & 11

Converting a virtual machine from Oracle VirtualBox to VMware Workstation Pro involves translating the virtual hardware and disk formats. The process is identical on both Windows 10 and Windows 11.

Here are the two best methods to achieve this, along with crucial post-conversion steps to prevent blue screens and driver conflicts.

Method 1: The OVF/OVA Export Method

Easiest & Recommended

This is the standard method that relies on the Open Virtualization Format (OVF), which is supported by both programs.

1

Prepare the VM in VirtualBox

  • Boot up your VM in VirtualBox.
  • Uninstall VirtualBox Guest Additions: Go to your VM's Control Panel → Programs and Features → Uninstall "Oracle VM VirtualBox Guest Additions".
    (If you skip this, your VMware VM might crash or have mouse/display issues).
  • Shut down the virtual machine completely (Do not pause or save state).
2

Export from VirtualBox

  • Open VirtualBox Manager.
  • Select your VM from the left pane.
  • Go to File > Export Appliance (or press Ctrl+E).
  • Select your VM and click Next.
  • Format Settings:
    • Format: Choose Open Virtualization Format 1.0 (OVF 1.0 is generally the most universally compatible with VMware).
    • File: Choose a destination path and save it as an .ova file (e.g., C:\VMs\MyMachine.ova).
  • Click Next, review the appliance settings, and click Export. Wait for the process to finish.
3

Import into VMware Workstation Pro

  • Open VMware Workstation Pro.
  • Go to File > Open (or press Ctrl+O).
  • Change the file type dropdown in the bottom right to All Supported Files or Open Virtual Machine Format (.ova, .ovf).
  • Browse to the .ova file you just created and open it.
  • Provide a name for the new VMware virtual machine and choose a storage path.
  • Click Import.

Note: If VMware shows an error saying the import failed because it "did not pass OVF specification conformance," simply click Retry or Relax. VMware will usually loosen its strict checks and import it successfully.

Method 2: Manual Disk Conversion

Fallback Method

If the OVA import fails or hangs, you can manually convert the VirtualBox hard disk (.vdi) directly into a VMware hard disk (.vmdk) using the Windows Command Prompt.

1

Find your VDI file

Locate the .vdi file for your VirtualBox VM. By default, this is usually in:
C:\Users\[YourUsername]\VirtualBox VMs\[VM Name]\

2

Convert VDI to VMDK via Command Prompt

VirtualBox includes a command-line tool called VBoxManage that can convert disk formats.

  1. Press Win + R, type cmd, and press Enter to open the Command Prompt.
  2. Navigate to the VirtualBox installation directory by typing: 
    cd "C:\Program Files\Oracle\VirtualBox"
  3. Run the clone command (replace the paths with your actual file paths): 
    VBoxManage.exe clonehd "C:\Path\To\Your\SourceDisk.vdi" "C:\Path\To\New\DestinationDisk.vmdk" --format vmdk
  4. Wait for the conversion to reach 100%.
3

Create a New VM in VMware

  • Open VMware Workstation Pro.
  • Go to File > New Virtual Machine.
  • Choose Custom (advanced) and click Next.
  • Select your hardware compatibility and click Next.
  • Choose "I will install the operating system later" and click Next.
  • Select the correct Guest OS (e.g., Windows 10, Ubuntu) that matches your converted disk.
  • Name your VM and choose its location. Continue clicking Next through the CPU and Memory settings, configuring them as needed.
  • Network Type: Choose NAT or Bridged depending on your preference.
  • Select a Disk: This is the crucial step. Choose "Use an existing virtual disk".
  • Browse to the .vmdk file you created in Step 2.
  • If VMware asks to convert the existing disk to a newer format, click Keep Existing Format (or convert it if you never plan to use it in older VMware versions).
  • Finish the wizard and boot the VM.

CRUCIAL Post-Conversion Steps

Regardless of which method you used, you must do the following once your VM boots up in VMware for the first time:

Install VMware Tools

Go to the VMware Workstation menu bar → VM > Install VMware Tools. Follow the on-screen prompts inside the guest OS. This replaces the drivers and fixes resolution, copy-paste, and network issues.

Reconfigure Network Adapters

Because the virtual MAC address and network adapter hardware have changed, Windows or Linux might identify it as a completely new network connection. You may need to re-enter static IP addresses if you were using them.

Reactivate Windows If applicable

If the guest OS is Windows, the massive hardware change (motherboard, CPU signature, network card) will likely trigger Windows Activation. You may need to run the Activation Troubleshooter inside the VM to tell Microsoft "I changed hardware on this device."
